# 箭头函数this的指向

  • 介绍

箭头函数this的定义：箭头函数中的this是在定义函数的时候绑定，而不是在执行函数的时候绑定。

一、 何为定义时绑定（看下面的例子）

let x = 11;
let obj = {
  x: 22,
  say: function () {
    console.log(this.x)
  }
}
obj.say() // 22

一般的定义函数跟我们的理解是一样的，运行的时候决定this的指向，我们可以知道当运行obj.say()时候，this指向的是obj这个对象。

let x = 11;
let obj = {
  x: 22,
  say: () => {
    console.log(this.x)
  }
}
obj.say() // 11

这个this指向的是外部的 x = 11。

所谓的定义时候绑定， 就是它会捕获其所在（即定义的位置）上下文的this值， 作为自己的this值，比如这里的箭头函数中的this.x，箭头函数本身与say平级以key:value的形式，也就是箭头函数本身所在的对象为obj，而obj的父执行上下文就是window，因此这里的this.x实际上表示的是window.x，因此输出的是11。(this只有在函数被调用，或者通过构造函数new Object()的形式才会有this)。

  • 类似的还有：
// 不使用箭头函数
let a = 11;
function test () {
  this.a = 22;
  let b = function () {
    console.log(this.a)
  }
  b()
}
new test() //  console.log(this.a) 11

// 使用箭头函数
let a = 11;
function test () {
  this.a = 22;
  let b = () => {
    console.log(this.a)
  }
  b()
}
new test() //  console.log(this.a) 22
  • 箭头函数中的this就是它会捕获其所在（即定义的位置）上下文的this值， 作为自己的this值。

二、注意

  1. 箭头函数作为函数的一种形式,对于this的处理和普通函数有所区别,其没有自己的this上下文,也就是说通过bind/call/apply函数方法设置this值时无效的，会被忽略。
  2. 因为箭头函数没有自己的this上下文，那么当它作为对象的方法函数运行时，this并不指向这个对象。
  3. 箭头函数中的this是指向外部作用域中的this。
  4. 你可以认为箭头函数的this和调用者无关，只和其定义时所在的上下文相关。
  5. 箭头函数没有this，所以也就不能用作构造函数。
  • 箭头函数转换成ES5
// ES6
function foo () {
  setTimeout(() => {
    console.log('id', this.id)
  }, 100)
}

// ES5
function () {
  var _this = this;
  setTimeout(function () {
    console.log('id', _this.id)
  }, 100)
}
